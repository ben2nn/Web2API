# 浏览器自动化功能

## 功能说明

浏览器自动化功能通过 Playwright/Camoufox 浏览器自动化操作 Qwen 网页版，模拟真实用户行为获取 AI 回复。当没有可用账号时，可以使用此功能作为备选方案。

## 工作原理

```
用户请求 → 网关(qwen2API) → 检查账号池
                ↓
        无可用账号？
                ↓
        启用浏览器自动化？
                ↓
        打开 Qwen 网页 → 输入问题 → 等待回复 → 抓取结果
```

## 配置方法

### 1. 后端配置

在 `.env` 文件中添加：

```bash
# 启用浏览器自动化
USE_BROWSER_AUTOMATION=true

# 浏览器超时时间（秒）
BROWSER_TIMEOUT=120
```

### 2. 前端配置

1. 打开系统设置页面
2. 找到"浏览器自动化"部分
3. 点击"启用"按钮

## 使用方法

### 自动使用

当启用浏览器自动化后，系统会：
1. 首先尝试使用账号池中的账号
2. 如果没有可用账号，自动使用浏览器自动化
3. 通过浏览器操作 Qwen 网页版获取回复

### 手动测试

```bash
# 激活 venv 环境
.venv\Scripts\activate

# 运行测试脚本
python test_browser_chat.py
```

## 实现细节

### 核心文件

- [browser_chat.py](backend/services/browser_chat.py)：浏览器自动化聊天服务
- [qwen_client.py](backend/services/qwen_client.py)：客户端，支持浏览器自动化
- [qwen_executor.py](backend/upstream/qwen_executor.py)：执行器，支持浏览器自动化
- [pool_acquire.py](backend/core/account_pool/pool_acquire.py)：账号池，支持浏览器自动化

### 浏览器自动化流程

1. **初始化浏览器**
   - 打开 Camoufox 反检测浏览器
   - 访问 Qwen 首页

2. **输入消息**
   - 找到输入框
   - 输入用户消息

3. **发送消息**
   - 点击发送按钮
   - 或按 Enter 键

4. **等待回复**
   - 检测回复开始
   - 轮询获取内容
   - 等待回复完成

5. **抓取结果**
   - 获取回复内容
   - 返回给用户

## 优缺点

### 优点

- ✅ 不需要登录 token
- ✅ 模拟真实用户行为
- ✅ 可以使用 Qwen 所有功能
- ✅ 不受 API 限制

### 缺点

- ❌ 速度慢（每次需要打开浏览器）
- ❌ 资源消耗大
- ❌ 需要处理网页变化
- ❌ 稳定性较低

## 故障排除

### 1. 浏览器初始化失败

**问题**: 浏览器初始化失败

**解决方案**:
```bash
# 检查 Camoufox 是否安装
python -m camoufox path

# 重新安装 Camoufox
python -m camoufox fetch
```

### 2. 找不到输入框

**问题**: 找不到输入框

**解决方案**:
- Qwen 网页可能更新了选择器
- 需要更新 `browser_chat.py` 中的选择器

### 3. 回复超时

**问题**: 等待回复超时

**解决方案**:
- 增加 `BROWSER_TIMEOUT` 配置
- 检查网络连接
- 检查 Qwen 网站是否正常

## 性能优化

### 1. 浏览器复用

当前实现每次请求都会打开新的浏览器窗口。可以优化为：
- 保持浏览器打开
- 复用浏览器会话
- 减少初始化时间

### 2. 并发控制

浏览器自动化消耗资源较大，建议：
- 限制并发数量
- 使用队列管理请求
- 监控资源使用

### 3. 缓存机制

可以添加缓存机制：
- 缓存常见问题的回复
- 减少重复请求
- 提高响应速度
